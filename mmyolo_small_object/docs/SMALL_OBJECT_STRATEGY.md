# 无人机小目标训练策略

这套工程的默认策略是先用最稳的方案把基线跑出来，再逐步上结构改造。

## 默认推荐

- 算法：`RTMDet`
- 数据：`大图切片 + 重叠滑窗`
- 分辨率：`1024` 或 `1280`
- 推理：后续建议补滑窗推理，和训练保持一致

原因：

- 无人机目标通常很小，切片带来的收益往往高于单纯换模型
- `RTMDet` 在 OpenMMLab 体系内更适合做稳定基线
- 先把召回和数据分布跑顺，再做结构创新，成本更可控

## 关于 YOLOv8-P2

`YOLOv8-P2` 适合极小目标，因为它增加了更浅层的高分辨率特征。

但这里应该把它视为第二阶段改造：

1. 先用 `RTMDet` 或普通 `YOLOv8` 跑稳定基线
2. 再扩展 `P2` 检测头
3. 最后做消融比较

## 关于 NWD Loss

对于像素很小的框，`NWD` 往往比纯 IoU 更稳。

但这类改动应该作为单独的自定义 loss 接入，不建议在第一版训练工程里硬写一个未验证实现。

推荐顺序：

1. `RTMDet + 切片 + 高分辨率`
2. 滑窗推理
3. `YOLOv8-P2`
4. `NWD Loss`
5. 注意力模块或 block 改造
