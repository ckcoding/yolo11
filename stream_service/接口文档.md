## 二、任务创建与处理接口

### 1. 启动视频流检测任务 (`POST /detect-stream`)
**描述：** 提交流媒体或本地视频，启动长时间视频推送分析。系统会自动开辟一个算力车位和任务线程。
**请求参数：**
```json
{
    "stream_url": "rtmp://192.168.3.231:1935/live/1",
    "alias": "demo",
    "mode": "webrtc",
    "names_dict": "drone_car:车辆,drone_person:人员",
    "confidence_display": false,
    "sample_fps": 15 
}
```
| 参数名 | 类型      | 必填 | 默认           | 说明 |
|---|---------|---|--------------|---|
| `stream_url` | String  | 是 | -            | 视频源地址 (RTSP/RTMP/HTTP-FLV) 兼容 `m3u8_url` / `url`。 |
| `names_dict` | String  | 否 | `""`         | `id:name,id2:name2` 字典映射，用于只检测指定分类。 |
| `alias` | String  | 否 | `"default"`  | 显示或管理使用的任务分组别名。 |
| `mode` | String  | 否 | `"stream"`   | 播放模式，可指定 `"stream"`, `"flv"`, `"webrtc"`, `"websocket"`。 |
| `confidence_display` | Boolean | 否 | `false` (默认false) | 若设为 `true`，则画面会在检测框上标注识别到的**类别名称**（默认 `false` 不标注文字，且不显示任何置信度）。 |
| `conf_thres` | Float   | 否 | `0.25`       | 检测过滤置信度阈值。 |
| `sample_fps` | int     | 否 | `15`（默认）     | **拆帧检测帧率**。控制每秒从视频流中抽取多少帧送入 AI 模型推理。设置越低 CPU/GPU 负载越小，设置越高检测越实时。有效范围 `1~120`，不传或传 `null` 时回落到服务器配置的全局 `output_fps`（默认 15）。 |

**响应成功 (200)：** 包含内部任务生成的 `task_id`、实际生效的 `sample_fps`，以及各类推流的转播地址 (`rtmp_url`, `m3u8_url`, `webrtc_url`, `flv_url`, `ws_url` 等)。
**响应示例：**
```json

        "device": "cpu",

        "target_classes": {
            "drone_car": "车辆"
        },
        "selected_models": [
            "drone_car"
        ],
        "pipeline": [
            {
                "id": "drone_car",

                "labels": [

            }
        ],
        "rtmp_url": "rtmp://127.0.0.1:1935/live/demo-721b7177-fb46-43da-9d24-d64596d5a1f2",
```

## 三、关闭与退出控制接口

### 1. 终止任务 (`POST /ai/stop`)
**描述：** 取消现有的推流或者推理分析计算，释放显存及队列承载量。
**请求参数：** `{"task_id": "all"}` 可填具体 `task_id` / 或 `alias` 名称，也可填 `"all"` 或者逗号隔开同时终止全部/多个。
**响应：** 返回 `stopped_count` 表示终止成功的分组数。
```json
{
    "code": 200,
    "message": "操作完成",
    "results": [
        {
            "task_id": "demo-f37e1e11-4a7c-4503-879f-2985d3b30eaa",
            "code": 200,
            "message": "任务 demo-f37e1e11-4a7c-4503-879f-2985d3b30eaa 停止"
        }
    ],
    "stopped_count": 1,
    "failed_count": 0
}
```

## 四、WebSocket 实时订阅接口

### 1. 服务端预警图文推送 (`WebSocket /ws/{task_id}`)
**接入方式：** 前端 JS 采用 WebSocket (`wss://` 或 `ws://`) 直连
**描述：** 专用于监控客户端（含手机 App）建立常驻的长连接。
- **初始任务状态快照**：连接建立后服务端立即主动推送一次 `type=task_info` 的 JSON，客户端无需轮询即可获取当前运行状态。
- **视频画面微流(Mode=websocket时)**：使用 Binary（Blob）解析推收 JPEG 字节帧。实际传输帧率由您设定的 `sample_fps` 决定，最高 30 FPS（两帧间隔 ≥ 0.033s）。
- **事件告警订阅**：检测到目标时服务端主动推送 `type=snapshot` 的 JSON。

**① 初始状态下推（连接时立刻收到，type = task_info）**
